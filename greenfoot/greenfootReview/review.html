Greenfoot Review

In this exercise, you are going to review some of the main aspects of making a game that you have learned while using Greenfoot.


Exercise 1 - Adding objects to a world

There are three ways to add objects to a world. Two of them involve creating objects within the Greenfoot environment by right-clicking on class's icon and selecting its constructor. However, these two methods are temporary and you have to keep adding the objects to the world. A more permanent method is to add code to the myWorld constructor.

  1. Add the following code to the MyWorld constructor that creates a Turtle object and adds it to the world at location (50, 50).

    Turtle sam = new Turtle();
addObject(sam, 50, 50);

  2. Add code to the MyWorld constructor that creates a Lobster object named larry and adds it to the world at location (50, 350).


Exercise 2 - move() and turn()

To move an actor within it's world you first used the move and turn methods. The move method moves an actor in its current direction or heading. The default direction or heading is 0 degrees. In Greenfoot, O degrees is located on the right edge of the world (East). The direction can be changed using the turn method.

  1. Add the following code to the Turtle class's act method so that it rotates sam the Turtle clockwise 1 degree each game cycle.

    turn(1);  // turn clockwise 1 degree

  2. Add code to the Lobster class's act method so that it rotates larry the Lobster counter-clockwise 1 degree each game cycle.

  3. Remove the turn methods added to the Turtle and Lobster classes.

  4. Add the following code to the Turtle class's act method so that it moves sam the Turtle across the world at a rate of one pixel per game cycle until he reaches the eastern or right boundary of the world.

    move(1);  // move 1 pixel in current direction

  5. Add code to the Lobster class's act method that will move larry the Lobster across the world at a rate of two pixels per game cycle until he reaches eastern boundary.

  6. Add the following code to the Turtle class's act method that turns sam the Turtle 180 degrees every time he reaches a world boundary.

    if(this.isAtEdge())
    turn(180);

  7. Add code to the Lobster class's act method so that larry the Loster turns 180 degrees every time he reaches a world boundary so that he will continue to move, but in the opposite direction.

  8. Include code that will stop larry from moving after he travels back and forth across the world 3 times.

  9. Modify the code in the Lobster class's act method so that larry turns 90 degrees instead of 180.

    Note:If you just do turn(90), then larry will still turn 180 degrees. The reason this occurs is because after he turns 90 degrees larry will still be on the boundary edge, therefore the if statement will be true for two game cycles or two calls to the act method.


Exercise 3 - Collision Detection

Greenfoot provides several methods for detecting Actor collisions. The method we used the most was the Actor class's isTouching method. The isTouching method returns true if the actor that made the call to isTouching collided with another actor.

  1. Add the following code to the MyWorld constructor.

    House house1 = new House();
House house2 = new House();
addObject(house1, 150, 150);
addObject(house2, 450, 150);
        
WalkingMan bob = new WalkingMan();
addObject(bob, 300, 150);

Exercise 4 - setLocation(), getX(), getY()

The second technique used to move actors is the setLocation method. Given the x and y coordinates of a location within the world the setLocation method will move the actor to this location.

  1. Add the following code to the MyWorld constructor.

    Ambulance amber = new Ambulance();
addObject(amber, 150, 250);

  2. Add the following code to amber the Ambulance's act method.

    setLocation(450, 250);

  3. Replace the setLocation method in amber's act method with the following code.

    setLocation(getX() + 1, getY());

    The getX() and getY() methods return the (x,y) location of an object. They can be used with the setLocation method to move objects relative to their current location. This code is basically saying, "Get my current x coordinate and increase it by one, then get my current y coordinate and leave it the same. Now move me to this location."


Exercise 5 - Keyboard Input

Greenfoot provides methods for both keyboard and mouse input. These methods are defined in a special class named Greenfoot. This class is special because the input methods are called directly by the Greenfoot class without the need of a Greenfoot object.

The methods isKeyDown and getKey are demonstrated below.

if(Greenfoot.isKeyDown("left"))  // Is the "left" key being pressed?
{
    // code not shown
}

String key = Greenfoot.getKey();  // get last key that was pressed and released
if("up".equals(key))              // Was that key equal to "up" key?
{
    // code not shown
}

  1. Replace the code in amber the Ambulance's act method with the following code.

    2. if(Greenfoot.isKeyDown("left"))
{
    setLocation(getX() - 1, getY());
    setRotation(180);  // west
}

else if(Greenfoot.isKeyDown("right"))
{
    setLocation(getX() + 1, getY());
    setRotation(0);    // east
}

    The setRotation method in this code turns amber so that she is facing in the direction specified. (East = 0, West = 180, South = 90, North = 270)

  2. Add code to amber's act method so that she can move up and down. The ambulance needs to rotated so that amber is facing the direction she is going.
