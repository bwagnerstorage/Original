Scanner Class

The Scanner class is a standard Java class that can be used to read user input from the keyboard or a file. The class contains several methods for reading user input from the keyboard. Right now we will only concern ourselves with two of its methods: nextInt() and nextDouble(). Each of these methods will be demonstrated below. Before we can use the Scanner class in our programs we must include the following import statement at the top of our program. 

import java.util.Scanner;

The import statement instructs the compiler to give a program access to the Scanner class. The words java and util in the statement are called packages and we will talk more about that later. The important thing to remember here is that if you want to use the Scanner class in your program you must include the above line of code at the very top of your program. Once we have imported the Scanner class into our program, we are ready to instaniate a Scanner object. Look at the following code: 

Scanner keyboard = new Scanner(System.in);

The above code creates a Scanner object named keyboard. The name keyboard is arbitrary, you can name your objects anything you want as long as you follow the rules for declaring identifiers. It should, however, be a name that actually describes what the object does. Since the Scanner object above is going to be reading user input from the keyboard, keyboard is a good choice for its name.

The System.in you see in the parentheses tells the computer that you want to read from the standard input device - the keyboard.

nextInt() Method

The nextInt() method inables users to read integer values from the keyboard. Look at the code segment below:

int num = keyboard.nextInt();

In this statement, the keyboard object responds to the message nextInt by waiting for the user to type a number and then press the Enter key, at which point the keyboard object passes the typed number on to the program. The number is then assigned to the variable num by means of the assignment operator (=). As the user types at the keyboard, the characters are automatically echoed in the console window, however it is not until the user presses the Enter key that this input becomes available to the program.

print Statement

It is customary to precede each nextInt in your program with a user prompt. A user prompt is simply a statement that informs the user of the type of information they will be entering from the keyboard. For example, in the code below, I instruct the user to "Enter the number of games played-->". This prompt tells the user what kind of information is required by the program to continue. 

System.out.
print
("Enter the number of games played--> ");    
// user prompt

int games = keyboard.nextInt();

The code above is introducing a new Java statement named print. You are already familiar with println statement. The difference between the print and the println statements is that the println moves the console cursor to the beginning of the next line after printing, whereas the print statement does not. Therefore when using a print statement the next item printed will be on the same line as the previous item.

The reason I used a print statement in the prompt, instead of the println, is so the user can enter the information on the same line as the prompt. Notice the output from the code above, the 5 entered by the user is on the same line as the prompt.

Enter the number of games played--> 5

nextDouble() Method

The nextDouble method is identical to the nextInt method except that it reads floating-point or decimal values from the keyboard instead of integers. Look at the following code segment:

Scanner keyboard = new Scanner(System.in);       
double num = 0;                                  
                                                 
System.out.print("Enter a decimal number --> "); 
num = keyboard.nextDouble();                     
// read decimal value from

                                                 
// the keyboard

System.out.println();
System.out.println("The number entered = " + num);

Here is a sample output:

Enter a decimal number --> 3.14159

The number entered = 3.14159

Notice that the value entered from the keyboard (3.14159) is a floating-point number. You are not required to include the decimal point when entering values with nextDouble(). If you enter, for example, 300, the nextDouble method will convert the number to 300.0 for you.
