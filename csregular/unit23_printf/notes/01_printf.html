printf

When writing data to a console window you have used both the print and println statements. Java offers another option for printing to the console window call printf (short for print formatting). This method provides alot more flexibility in how information is displayed on the screen. In this unit you will learn how to use the printf method to display information on the console window.

printf Form

The way that a printf statement is constructed is different from the way you would do a print and println. Here is the form of the printf statement:

A printf statement has two parts: the formatting string and the data to be printed. The formatting string uses special commands to describe how the data will appear on the screen. The number of data items that a printf statement can print is virtually limitless. To distinguish between each data item the printf uses commas.

Formatting String

The formatting string uses special commands to describe how the data will appear on the screen. A command is made up of several parts. Look at the illustration below.

  • % sign - the percent sign must precede all formatting commands.
  • Format Flags - used to apply special formatting to the data. We will limit our dicussion to only one flag, the hyphen (-). The hyphen flag is used to cause the data to left align, instead of right align, which is the default.
  • Column Width - specifies how many columns on the screen the data will occupy.
  • Format Type - specifies what type of data the command refers (int, float, string, etc.). There are many format types. We will limit our discussion to only four:
    Format Type Description
    d integer (decimal)
    f floating-point
    s string
    n newline

Format Type - n

The n format type is used to embed newline commands within the formatting string.

The following example

  System.out.printf("%5d", 100);
  System.out.printf("%5d", 200);

produces this output

     100     200

To make the two numbers appear on separate lines you have to add a %n command

  System.out.printf("%5d
%n
", 100);
  System.out.printf("%5d", 200);

which produces this output

     100
     200

or

  System.out.printf("%5d
%n%n
", 100);
  System.out.printf("%5d", 200);

which produces this output

     100
	 
     200

Labeling Data

Suppose you want to print a name and grade entered from the keyboard using the following format

Name =    John Wayne
Grade =          100

where the person's name and grade are lined up on the right in a columnar format (right justified).

Since the name and grade can be varying lengths you know it is not possible to do this using only print and println. So you write the following code using printf

String name = keyboard.nextLine();
int grade = keyboard.nextInt();
System.out.printf("Name = %12s%n", name);
System.out.printf("Grade = %5d%n", grade);

which produces the following output (Assuming you entered "John Wayne" for the name and 100 for the grade)

Name =   John Wayne
Grade =      100

Since the length of the name and grade are not known until entered by the user you realize that the column width specifiers 12 and 5 won't work for any name and number. So you go back to the drawing board and write the following code

String name = keyboard.nextLine();
int grade = keyboard.nextInt();
System.out.printf("%-8s%12s%n", "Name = ", name);
System.out.printf("%-8s%12d%n", "Grade = ", grade);

which produces the following output

Name =    John Wayne
Grade =          100

After several tests you come to the conclusion that this code will work as long as the name or grade entered by user is not larger than 12 characters. You can now sleep peacefully knowing your job is done.
