Compound Conditions

Conditions come in two forms, simple and complex. A simple condition is a Boolean expression that does not contain any other Boolean expression. In the Jeroo language, a simple condition is formed by invoking a single sensor method. A complex condition is created by using logical operators to combine conditions. The Jeroo language contains the three most commonly used logical operators: negation (not), conjunction (and), and disjunction (or). Jeroo's Java-style language uses special keystrokes for each of these as shown in the table below.


Negation (NOT Operator)

The negation reverses the value of a Boolean expression, changing true to false and false to true. The table below shows the result of applying the negation operator to a Boolean expression. In this table, P represents an arbitrary Boolean expression. The two rows underneath P show its possible values. The second column shows the corresponding values for the expression !P.


Examples

if(!bob.isNet(AHEAD))  // if not net in front of Jeroo
{
   bob.hop();
}

while(!bob.isWater(AHEAD))  // while not water in front of Jeroo
{
   bob.plant();
   bob.hop();
}

Conjunction (AND Operator)

The conjunction combines two Boolean expressions, to create a third that is only true when both of the original expressions are true. The next table shows the result of applying the conjunction operator to a Boolean expression. In this table, P, and Q represent arbitrary Boolean expressions. The rows underneath P and Q show all possible combinations of their values. The third column shows the corresponding values for P && Q.


Examples

if(bob.isNet(AHEAD) && bob.hasFlower())  // if net ahead and Jeroo has a flower
{
   bob.toss();
   bob.hop();
}

while(bob.isFacing(NORTH) && !bob.isWater(AHEAD))  // while facing North 
{                                                  // and not water ahead
   bob.hop();
}

Disjunction (OR Operator)

The disjunction combines two Boolean expressions, to create a third that is only false when both of the original expressions are false. The table below shows the result of applying the disjunction operator to a Boolean expression. In this table, P, and Q represent arbitrary Boolean expressions. The rows underneath P and Q show all possible combinations of their values. The third column shows the corresponding values for the expression P || Q.


Examples

if(bob.isNet(AHEAD) || bob.isJeroo(LEFT))  // if net ahead or Jeroo to the left
{
   bob.turn(RIGHT);
}

while(!bob.isWater(AHEAD) || !bob.isNet(AHEAD))  // while not water or net ahead
{
   bob.plant();
   bob.hop();
}
