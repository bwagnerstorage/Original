Variable Scope

Variable scope identifies which parts of a program can see or use a variable. A variable exists from the point in a program where it is declared to the end of the block that contains the declaration.

The illustration below explains variable scope.

Initializing Instance Variables

From the illustration above you learned that instance variables have class scope. Instance variables are unique in that they do not have to be initialized. Recall that when you initialize a variable you are both declaring it and giving it an initial value.

int num = 25;   
 // initialized with a value of 25

If you declare an instance variable without initializing it the JVM(Java Virtual Machine) will assign the variable its default value. Look at the example below.

private int sum;

Since the instance variable sum is declared without being initialized the JVM will assign it a default value of zero. The table below shows the default values for the primitive data types we use in this course.

Data Type Default Value
int 0
long 0
double 0.0
boolean false

If an instance variable is an object variable, that is, a variable that stores a reference to an object, it is assigned a default value of null by the JVM if it not assigned to an object. Here is an example.

private Scanner scan;

Since instance variable scan is not assigned a reference to an object it is assigned a value of null by the JVM.

Initializing Local Variables

Unlike instance variables, local variables that have not be initialized are not assigned default values by the JVM. Look at the following example.

public void method1()
{
    int sum;
}

Since the local variable sum is not initialized, it is not assigned a value. A local variable that has no value can cause a compiling error in some situations. Look at the following example.

public void method1()
{
    int sum;
	
    sum += 5;
}

This code will produce the following compiling error.

error: variable sum might not have been initialized

This code causes an error because the += operator needs take the value that is currently in sum and add 5 to it but it can't because sum does not have a value. The code can be fixed by initializing the variable sum to zero.

public void method1()
{
    int sum = 0;
	
    sum += 5;
}
Programming Note: It is a good programming practice to always initialize local variables.

Initializing String Variables

When initializing a String variable, if you want to assign it a default value, use the empty string "" (no space between quotes).

String str = "";      
// initialized to the empty string
