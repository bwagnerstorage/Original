Jeroo Simulator

According to the story on jeroo.org, the Jeroo programming environment was developed as a simulator, so that scientists could, in the safety and comfort of their lab, study the peculiar behaviors of these mysterious creatures. The scientist knew that Jeroos were similiar in nature to kangaroos, but they also knew they possessed their own unique set of behaviors. For example, the fact that Jeroos only turn right or left in 90 degree increments. The scientist hoped the simulator might help shine some light on why these fascinating animals behave the way they do.

In the Jeroo simulator, we learned that in order to add a Jeroo to an island environment we had to use code like the following:

Jeroo bob = new Jeroo();

In programming terminology, Jeroo is referred to as a class. Classes are the backbone of Object-Oriented Programming(OOP) languages, because they are responsible for defining the attributes and behaviors for the objects they create. Referring again to the line of code above, bob is a special type of variable called an object variable or more simply just an object . This object, referred to as bob, is "born" into this programmming world with all the atttributes and behaviors described by its designer, the Jeroo class.

What attributes and behaviors did the Jeroo class give to bob? The attributes include the ability, like all Jeroos, to store flowers in his pouch, as well as the ability to know which direction he is facing and where he is located on the island. The behaviors include the abilty to hop and turn, as well the ability to pick, plant, give, and toss flowers. There are others we will explore in a later unit.

Jeroo Methods

In Java, the behaviors defined by a class are called methods. A method can be defined as a name given to a block of code that can be executed by calling the method by its name. Look at the following code,

Jeroo bob = new Jeroo();
bob.hop();

In this example, the Jeroo class's hop method is being invoked (called) by the object bob.

One of the cool features in the Jeroo simulator is that it allows programmers to make their own Jeroo methods. Follow the directions below to see how.

Directions

  1. Click on this link to open the Jeroo environment in your browser.

  2. Click on the button START CODING.

  3. Copy the following starter code to the Main Method editor.

    method main()
{
  Jeroo bob = new Jeroo(5,5,EAST,50);
       

}

  4. To add your own method to Jeroo class you first have to switch to the Jeroo Methods pane. To do this click the mouse on the Jeroo Methods tab.

  5. With the Jeroo Methods pane visible. Let's make a method. One of things that I don't like about digital Jeroos is they don't know how to turn around, so let's fix that by making a method that tells the Jeroos how to turn around.

        Step 1: Type: method turnAround() and then open and close curly brackets.

        Step 2: Inside the curly brackets type: turn(RIGHT);

        Step 3: Copy the previous code and paste in the method so that there are two turn rights.

    Notice: We did not use a Jeroo object to call the turn methods. This is because we are writing code that is actually being added to the Jeroo class. This new method will be available to all Jeroo objects, not just bob.

  6. Now that we have made our own method. Let's see if bob has really learned how to turn around. Click on the Method Main tab to go back to the main method and add the following code: bob.turnAround();

  7. Now that you know how a Jeroo method is made try making one of your own.
