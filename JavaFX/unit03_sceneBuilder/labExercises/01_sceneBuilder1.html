Scene Builder

Coding a robust GUI interface can be a daunting task. Fortunately there is an application named Scene Builder that can help simplify this task. Oracle describes Scene Builder like this,

JavaFX Scene Builder is a visual layout tool that lets users quickly design JavaFX application user interfaces, without coding. Users can drag and drop UI components to a work area, modify their properties, apply style sheets, and the FXML code for the layout that they are creating is automatically generated in the background. The result is an FXML file that can then be combined with a Java project by binding the UI to the applicationâ€™s logic.

Drag-and-Drop UI Controls

With Scene Builder, you have the ability to build a GUI interface by simply dragging and dropping layouts and controls onto your scene's window.

Select the Controls Panel on the side bar to the left. These are all of the UI controls currently available in JavaFX.

Drag-and-drop the Button control onto your scene window. When a control is added to a scene's window Scene Builder automatically updates the associated fxml file.

The side bar to the right has three panels: Properties, Layout, and Code. Click on the Button control to select it. It is selected if you see blue handles around its perimeter. Now click on the Properties panel to expand it.

Scene Builder gives you access to all of a control's properties. Again any changes made here will automatically be reflected in the fxml file. For example, change the Text property value to word "Go". This will change the name displayed on the button. You can also change a button's name by double clicking on the button itself.

Click on the Layout panel.

Two of the most common properties to modify here are the preferred width and preferred height properties. For example, change the Pref Width value to 100 and change the Pref Height value to 30. This will increase the size of the button. When displaying a scene's window JavaFX will try to honor the preferred width and preferred height properties when displaying the controls, however that all depends upon which type of layout(i.g. AnchorPane, VBox, HBox, etc.) they were placed.

Click on the Code panel.

Automatic Creation of Controller Class

Another benefit of Site Builder is that it can create your program's Controller class. It does this by taking the value assigned to each control's fx:id property and creating a variable in the Controller with that name. If you do not assign a value to the fx:id property then a variable will not be created for that control. Recall that when your program calls the FXMLLoader class's load method it will inject each variable in the Controller class with the object reference of the variable's corresponding control.

Back to the left side bar there is one more panel I would like to discuss. Under the Document section there is a panel named Hierarchy, click on it.

This panel displays the current scene's scene graph. Sometimes you may find it easier to drag-and-drop controls onto the scene graph instead of the scene window. For example, click on the Label control and drag-and-drop it onto the scene graph.

Generate Controller class

  1. Open the JavaFXExercise4 project in NetBeans, if not already open.

  2. Right-click on the file FXMLDocument.fxml and select Edit. Scroll through the file and notice that all of the controls that you added through Site Builder are coded there.

  3. Open the file FXMLDocumentController. Noticed that the file only contains one method named initialize.

  4. Right-click on the file FXMLDocument.fxml again, This time select the option Make Controller.

  5. View the file FXMLDocumentController again.

  6. The next step is to add code in the initialize method to add data to the controls and to implement the event handlers. To speed up this process I have provided the Controller class code below. Delete the code currently in the Controller class and replace it with this code.

    package javafxexercise4;

  import javafx.collections.FXCollections;
  import javafx.collections.ObservableList;
  import javafx.event.ActionEvent;
  import javafx.fxml.FXML;
  import javafx.scene.control.ComboBox;
  import javafx.scene.control.Label;
  import javafx.scene.control.ListView;
  import javafx.scene.control.RadioButton;
  import javafx.scene.control.TableColumn;
  import javafx.scene.control.TableView;
  import javafx.scene.control.TextField;
  import javafx.scene.control.ToggleGroup;
  import javafx.scene.control.cell.PropertyValueFactory;
  import javafx.scene.input.KeyCode;
  import javafx.scene.input.KeyEvent;
  
  public class FXMLDocumentController {
      
      @FXML
      private Label label;
      @FXML
      private ComboBox<String> myComboBox;
      @FXML
      private ListView<String> myListView;
      @FXML
      private TableView<Candidate> myTable;   // Candidate class defined below
      @FXML
      private RadioButton yesRadioButton;
      @FXML
      private RadioButton noRadioButton;
      @FXML
      private TextField myTextField;
      @FXML
      private TableColumn<Candidate, String> nameTableColumn;
      @FXML
      private TableColumn<Candidate, String> votesTableColumn;
     
      // event handlers
      @FXML
      private void handleButtonAction(ActionEvent event) {
          System.out.println("You clicked me!");
          label.setText("Hello World!");
      }
      @FXML
      private void handleTextFieldChange(KeyEvent event) {
          if(event.getCode() == KeyCode.ENTER)
          {
              String str = myTextField.getText();
              myListView.getItems().add(str);
              myTextField.clear();
          }
      }
      
      // called after component variables injected from FXML file
      public void initialize() {
          
          // radio buttons
          ToggleGroup radioGroup = new ToggleGroup();
          yesRadioButton.setToggleGroup(radioGroup);
          noRadioButton.setToggleGroup(radioGroup);
          
          // combo box
          ObservableList<String> list;
          list = FXCollections.observableArrayList("Monday", "Tuesday", "Wednesday", "Thursday", "Friday");
          myComboBox.setItems(list);
          myComboBox.getSelectionModel().selectFirst();   // assign default value
          
          // list view
          list = FXCollections.observableArrayList("horse", "cow", "pig", "goat", "chicken");
          myListView.setItems(list);
          
          nameTableColumn.setCellValueFactory(new PropertyValueFactory<>("name"));
          votesTableColumn.setCellValueFactory(new PropertyValueFactory<>("votes"));
          
          // table view
          ObservableList<Candidate> list2 = FXCollections.observableArrayList();
          list2.add(new Candidate("Gomez", 20));
          list2.add(new Candidate("Smith", 8));
          list2.add(new Candidate("Choo", 2));
          list2.add(new Candidate("Harris", 35));
          list2.add(new Candidate("Salazar", 52));
          list2.add(new Candidate("Li", 14));
          list2.add(new Candidate("Adams", 26));
          list2.add(new Candidate("Kyle", 33));
          list2.add(new Candidate("Lucas", 5));
          list2.add(new Candidate("Howard", 16));
          list2.add(new Candidate("Riser", 25));
          
          myTable.setItems(list2); 
      }
      
  }
