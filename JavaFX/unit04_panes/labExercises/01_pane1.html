Layout Panes

JavaFX provides a set of classes called panes. The purpose of these panes are to provide an easy way to manage the user interface of a JavaFX application. These panes are containers for storing UI controls, as well as other panes, so that the controls can be arrange on the scene in an organized visually appealing manner.

A JavaFX application can manually lay out the UI by setting the position and size properties for each control. However, an easier option is to make use of layout panes. JavaFX provides several layout panes for the easy setup and management of classic layouts such as rows, columns, stacks, tiles, and others. As a window is resized, the layout pane automatically repositions and resizes the nodes that it contains according to the properties of the individual nodes.

In this unit, I am not going to discuss the ends and outs of each layout pane, I will provide links below that can provide this information, instead I am going to demonstrate the advantage of placing layout panes inside other layout panes. You will see that by nesting layout panes even more refined positioning of elements on a scene can be achieved by taking advantage of the layout properties provided by each pane.

To learn more about layout panes click on one of the following links:

AnchorPane

Before we start this exercise I would like to take a look at a pane that we have already used several times in our exercises, namely the AnchorPane. Although we have used this pane, we have never actually discussed the layout properties of this container. An AnchorPane is divided into four regions: North, South, East, and West. When nodes (i.g. controls, containers) are placed within a pane they are attached to one or more of these regions. If a node's region or regions is not explicitly assigned, then by default, the AnchorPane class attaches the node to both the North and West regions. The effects of these attachments is best understood by looking at a demonstration.

  2. Copy the AnchorPane folder in the zip file to your work folder.

  3. Open the project AnchorPane in NetBeans.

  4. Double-click on the file FXML_AnchorPane.fxml to open it in Scene Builder. You should see the following scene window.

    Looking at Hierarchy panel you will notice that there is an AnchorPane at the root level of the scene graph and it has three children: a Button, a Label, and a TextArea.

  5. Click on the Preview Menu on the menubar and select Show Preview in Window

  6. Position the mouse over the right edge of the window so that the mouse cursor changes to a double arrow. Now click the left mouse button and drag the mouse to enlarge the size of the window horizontally. Notice how the UI controls responded to the resizing. Now do the same process for the other three edges of the window. Which two window edges (regions) had an effect on the position of the controls? If you said Top (North) and Left (West) then you are correct. Since I did not explicitly assign a region to any of the controls they were automatically attached or glued to the North and West regions.

  7. Close the preview window.

  8. Click on the Button control to select it. Then toggle the Layout panel. You should see the Anchor Pane Constraints controller at the top of the panel.

    This controller is where you attach nodes to the different regions of the AnchorPane.

  9. Click on the horizontal bar that extends from the square in the middle of the controller. This will highlight the controller in red like the image below.

    This will attach the Button to the AnchorPane's East region. The number that appears in the textbox is the distance the Button's right edge is from the right edge of window.

  10. Open a preview Window and resize the window's right edge to see the effect this attachment has on the button.

  11. While the button is still attached to the East region, use the controller to also attach the button to the West region.

  12. Open a preview Window and resize the window again to see the effect.

  13. Now play around with the controller and the other UI controls until you have a good understanding of how the AnchorPane works. More than one control can be attached to the same region or regions.

By placing the GridPane container inside an AnchorPane container we are able to position the grid exactly where we want it and it will be anchored or glued to that location.
