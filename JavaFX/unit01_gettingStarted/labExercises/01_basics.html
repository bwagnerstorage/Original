Notice the @Override declaration above start's method header. This is an example of Java annotation, this particular annotation insures that the method being overridden, in this case, start, is actually a super class method, if the method is not in a super class because it is misspelled for example, an error would occur at compile time instead of run time. And as you know compile errors are much easier to debug than runtime errors. Annotations are not always required, however, it is consider a good programming practice.

In a standalone (desktop) application the start method is executed in the main method by calling another Application class method named launch.

The Theater

The start method includes a parameter of type Stage named primaryStage. This terminology is borrowed from the theater. The stage in the theater is where the play is performed and viewed by the audience. In JavaFX the stage is a window where the GUI program is executed and viewed by the user.

Stage

A scene in the theater is a set of activities that happen at a particular time on the stage. A scene in JavaFX is a set of GUI controls or widgets that are on the stage at a particular time. These include buttons, text fields, menus, and combo boxes, just to name a few.

Building a JavaFX Application

The process of building a JavaFx application is fairly straightforward. Your first task is to create the controls your program will need to interact with the user, next you create a scene and add the controls, and lastly you assign the scene to a stage.

Our HelloWorld program only had one control, a button. Instead of being added directly to a scene it was first added to a StackPane and the StackPane was then added to the to the scene. A StackPane helps with the layout or positioning of controls in a scene. You will learn more about panes in a future lesson.

StackPane root = new StackPane();
root.getChildren().add(btn);
			   
Scene scene = new Scene(root, 300, 250);
			   
primaryStage.setTitle("Hello World!");
primaryStage.setScene(scene);
primaryStage.show();

This code is pretty self explanatory.However, there is one concept in this code I would like to look at and that is scene graphs.

Scene Graph

In computer science a scene graph is data structure used to organize and manage a variety of media types, most notably 3D graphics and games. JavaFX uses scene graphs to organize all the components or controls added to a scene.

The scene graph stores the whole scene in the form of a graph of connected objects called nodes. There are two types of nodes: branch nodes and leaf nodes. A branch node is a node that can contain other nodes (child nodes). Branch nodes are also referred to as parent nodes because they can contain child nodes. A leaf node is a node which cannot contain other nodes. The root node is the first node in the graph. The root node does not have a parent.

StackPane root = new StackPane();
root.getChildren().add(btn);
Scene scene = new Scene(root, 300, 250);

In the third line of code a scene object is created. It's constructor assigns the root of its scene graph to the StackPane object named root. It is also sets the initial dimensions of the stage's window to 300 x 250. In the second line of code the StackPane object, root, gets a list of its current children, then it calls an add method to add the button to this list, which adds the button to the scene graph as a child of StackPane.
