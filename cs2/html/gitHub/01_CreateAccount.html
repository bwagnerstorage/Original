What is GitHub

To understand what GitHub is you first have know what a Git is. A Git is a version control system to keep track of changes to files and projects over time. If you use a git it will allow you to make changes to a file but still keep previous versions of the file in case you want/need to revert back to one. It works like undo and redo in an editor, but it does it for entire files.

Now what is GitHub? GitHub is a website that hosts Git repositories online, allowing software developers to store their programming projects on the cloud with version control(Git). GitHub also makes it easier for developers to share their code with other developers. Founded in 2008, GitHub is by far the largest website of its kind. It is currently owned by Microsoft.

This begs the question what is a repository? A repository or "repos" are folders which contain snapshots or previous versions of the files in the folder. These repos are intentional made by developers so that when they are working on a project, if they make changes to a file or files in that project they can be uploaded or "committed" to GitHub so that GitHub can track those changes.

That was a very brief description of GitHub. To be honest, it's a lot easier for someone to understand what GitHub can do through demonstration. So let's create your own GitHub account so you can see what GitHub can do.

GitHub Image Repository

You can upload any file type to a GitHub repository. This includes image files. The reason that I had you create a GitHub account so early in this course is because we are going to use the fact that GitHub can store images to our advantage. One of the disadvantages of using online editors for free is they typically do not allow you to upload any files to your projects or pens. This creates a problem in that in order to be able to display images in a web page the page needs access to the image file. In the next section, I'm going to walk you through the steps used to upload a file to a repository, in this case an image file.

Uploading Image Files to a GitHub Repository

  1. Go to the GitHub website.

  2. Click on the "Sign in" button in top right corner of the window. When the Sign in window appears sign in to your account using the information you provided when you signed up.

  3. If you signed in correctly, your accounts startup page will appear. On the left side of the page next to the label named "Repositories" you will see a green button labelled "New". Click that button.

  4. The next page entitled "Create a new repository" has a text field labelled "Repository name". In this box, type the word "images".

  5. Now scroll to the bottom of the page and click on the green button labelled "Create repository". You have just created your first GitHub repository named "images".
