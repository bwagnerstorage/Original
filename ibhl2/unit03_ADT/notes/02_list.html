List ADT

A List ADT is an abstract data type that defines data structures that are ordered by position. Each item except the first has a unique predecessor, and each item except the last has a unique successor.

In Java, the List ADT is implemented through the List interface. The List interface adds position-oriented operations, including the ability to insert an element, get an element, as well as remove or change an element.

List Interface (partial listing)

interface java.util.List<E>

int size()  
boolean add(E obj) appends obj to end of list; returns true
void add(int index, E obj) inserts obj at position index
E get(int index)  
E set(int index, E obj) replaces the element at position index with obj; returns the element formerly at the specified position
E remove(int index) removes element from position index, moving elements at position index + 1 and higher to the left (subtracts from their indices) and adjusts size; returns the element formerly at the specified position
Iterator <E> iterator()  
ListIterator <E> listIterator()  

Two classes implement the List interface: ArrayList and LinkedList. Since both of these data structures implement the List interface their behavior is similar.

Note: There seems to be some confusion in the programming community as to the difference between a data structure and a collection. The IB documentation defines a collection as "a set of elements. The elements may be of any type (numbers, objects, arrays, Strings, etc.)". From this definition, any class that implements a data structure is a collection.

Static Data Structures vs Dynamic Data Structures

From a users point of view you would think that the ArrayList data structure and the LinkedList data structure are the same. This is true for what operations the data structures perform but how theses operations are implemented (under the hood) is completely different.

The ArrayList class is implemented using an array. An array is what is called a static data structure. A static data structure is a data structure that has a fixed size. This means once memory has been allocated for the structure it cannot be change while the program is running. Data is accessed in a static structure through an index.

The LinkedList class is implemented as a dynamic data structure. A dynamic data structure has a flexible size, it can grow or shrink as needed to contain the data to be stored. Data is accessed in a dynamic structure through a reference or pointer.

The table below shows some of the differences between the two data structures.

Static Data Structures Dynamic Data Structures
Fast access to elements Slower access to the elements
Expensive to insert/remove elements Fast insertion/deletion of elements
Fixed maximum size Flexible size
Memory is reserved at the time of compilation Memory allocation takes place at the run time, only required amount of memory is allocated

A data structure can be implemented as either a static or dynamic structure. Deciding on what type to use depends on what the data structure is design to do. Most abstract data structures are implemented as dynamic data structures.

Linked List Data Structure

A linked list is a dynamic data structure where each element is a separate object, called a node, that contains data and a link to the next node in the list.

A circular linked list is a linked list in which the last node points to the first node.

A good example of an application where circular linked list should be used is a timesharing problem solved by the operating system. In a timesharing environment, the operating system must maintain a list of present users and must alternately allow each user to use a small slice of CPU time, one user at a time. The operating system will pick a user, let him/her use a small amount of CPU time and then move on to the next user, etc. For this application, there should be no null pointers unless there is absolutely no one requesting CPU time.

A doubly linked list is a linked list that contains links to next and previous nodes. Unlike singly linked lists where traversal is only one way, doubly linked lists allow traversals in both directions.

Iterators

Java provides three methods or techniques for traversing the contents of a list (that is, to visit each of the items in the collection in turn). The first approach is to use methods from the List interface such as size and get. The second and more functional approach is to use a mechanism called an iterator. The third is to use the enhanced for loop.

An iterator is an object associated with a collection that makes it possible to traverse the collection. To provide consistence between all of the collections in the Java Collections Framework (JCF), code written using iterators will work for any type of collection (linear, hierarchical, etc).

An iterator's current position is always in one of three places:

  • Just before the first item
  • Between two adjacent items
  • Just after the last item

Initially, when an iterator is first instantiated, the position is immediately before the first item in the list. From this position the user can navigate to the next position in the list.

Iterator Methods

boolean hasNext() returns true if there are any items following the current position
E next() Returns the next item and advances the position
Precondition: hasNext returns true.
Throws a NoSuchElementException if hasNext returns false
void remove() Removes the item returned by the most recent call of next
Precondition: next has recently been called.
Throws an IllegalStateException if the next method has not yet been called or if the remove method has already been called after the last call to the next method

Using Iterator

You create an iterator object by sending the message iterator to a collection as show in the following code segment:

List <String> myList = new LinkedList<>();
Iterator<String> iter = myList.iterator();   // get iterator from list

Once an iterator object has been retrieved from a list the methods hasNext and next are used for navigating the list. The method remove will mutate the underlying list.

The code segment below demonstrates a traversal of a list to display its contents using an iterator:

while(iter.hasNext())
{
   System.out.println(iter.next());
}

Using Enhanced for Loop

The enhanced for loop is a condensed version of an iterator block. Refer to the samples below.

// Enhanced for loop version
for(String str : myList)
{
   System.out.println(str);
