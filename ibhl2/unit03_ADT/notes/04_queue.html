QUEUE ADT

Like stacks, queues are linear collections. However, insertions in a queue are restricted to one end, called the rear, and removals are restricted to the other end, called the front. A queue thus supports a First-In, First-Out protocol (FIFO). Queues occur in everyday life from checkout lines in stores, to highway tollboth lines, and airport baggage check-in lines. A queue is used whenever people or things are lined up for processing on a first-come, first-serve basis.

Queues have two fundamental opertions: add, which adds an item to the rear of a queue, and remove, which removes an item from the front. (The traditional names of these operations are enqueue and dequeue)

Special Implementations of a Queue

Priority Queue

    A priority queue is a collection of elements that each element has been assigned a priority such that order in which elements are deleted and processed comes from the following rules:

  • An element of higher priority is processed before any element of lower priority.
  • Two element with the same priority are processed according to the order in which they were added to the queue(FIFO behavior).

A good example of a priority queue is the emergency room at a hospital. As patients enter the emerency room they are placed on a waiting list queue, however patients with more serious injuries are moved to the front of the list.

Circular Queue

A circular queue is a queue with a finite capacity (limited to a fixed number of items). When the number of items in the circular queue is less than the capacity, it behaves as a normal queue. However, when the queue is full, new items added to the queue overwrite the older items.

A good visual image of a circular queue is a merry-go-round.

Applications of Queues

    Applications of queues in computer science are numerous. Following are just a few examples:

  • Multitasking operating system - operating systems often maintain a queue of processes that are ready to execute or that are waiting for a particular event to occur.
  • Buffer - computer systems must often provide a "holding area" for messages between two processes, two programs, or even two systems. This holding area is usually called a "buffer" and is often implemented as a queue.
  • Storing client service requests in a web server
  • Printer queue - printers store job requests in queues so that first job sent to the printer is first job printed.
  • Simulations - programs that attempt to model real-world phenomenon.

Java Collections Framework(JCF)

The developers of the JCF decided not to include a Queue class in the framework, instead to create a queue you use the LinkedList class which implements the Queue interface. Because of this implementation decision the add and remove methods of the LinkedList class are used instead of the more traditional queue operations enqueue and dequeue.

interface java.util.Queue<E>
boolean add(E obj) enqueues obj at the end of the queue; returns true
E remove() dequeues and returns the element at the front of the queue
E peek() returns the element at the front of the queue; null if the queue is empty
boolean isEmpty() returns true if queue is empty; false otherwise

Using a Queue

To create a queue use the LinkedList class which implements the Queue interface.

Creating a Linked List Object 
LinkedList<Integer> queue = new LinkedList<>();  // create queue

or

List<Integer> queue = new LinkedList<>();        // create queue

Adding an Object to a Queue

queue.add(57);  // adds item to rear of queue

Removing an Object from a Queue

int num = queue.remove();  // removes item from front of queue

Viewing the Top of a Queue

int num = queue.peek();  // does not remove the item

Traversing a Queue

while(!queue.isEmpty())
{
   System.out.println(queue.remove());
