Abstract Data Type(ADT)

One of the essential features of a computer is its ability to store and manipulate data. Computer programs often need to store large amounts of data and one way to accomplish this is to use an array. An array is very useful, however, it has many deficiencies, one of which is its requirement of a fixed size. For this reason, computer scientists have developed other ways to store data in programs called Abstract Data Types.

An Abstract Data Type(ADT) is a description for a collection of data and a set of operations that act on the data. These data types are abstract because they only describe what can be done to the data not how it is done. The actual implementation of these ADTs is done through the construction of data structures. A data structure is the physical implementation of an ADT. In Java, these data structures are implemented as classes.

An ADT describes the common actions/methods of a data structure but the implementation details are not known. That is left to the implementing data structure or class.
Common Operations

There are three common operations that all ADTs share. They include:

  • Add - add a data item to a collection.
  • Remove - remove a data item from a collection.
  • Retrieve - get a data item from a collection.

How theses are operations are performed by the different ADTs depends on their implementation.

Below is a list of some of the most common ADTs:

  • List
  • Bag
  • Set
  • Stack
  • Queue
  • Priority Queue
  • Tree
  • Heap
  • Hash
  • Map
  • Graph

For example, the List ADT describes a data structure that contains an ordered collection of values, where the same value may occur more than once. In Java, ADTs are represented through interfaces. Below is the partial specification for Java's List interface:

public interface List
{
    int size()
    boolean add(E obj)
    void add(int i, E obj)
    E get(int i)
    E set(int i)
    E remove(int i)
}

This interface fullfills the specification for a List ADT by requiring any class that implements this interface to have an order collection of data that can be added, removed, retrieved, and updated. Two of the classes(data structures) that implement the List interface are ArrayList and LinkedList. Both classes provide the operations specified in the List interface but how they store the data in the class and carry out the operations is done in two completely different ways.

Collection

A collection is a group of items forming a conceptual unit. In computer science collections are ADTs that have been grouped together. These ADTs are implemented through data structures. A collection is essentially a collection of ADTs.

Below is a list of common collection categories. These categories describe how the data is organized in the collections:

  • Linear: Arrays, Lists, Stacks, Queues, ...
  • Hierarchical: Heaps, Trees, Hashes, ...
  • Connected: Graphs
  • Unordered: Sets, Bags, Maps, ...

Collection classes play an important role in almost every non-trivial program written today. For this reason most high-level languages provide collections through class libraries including the Java Collection Framework(JCF), C++ Standard Library, and .NET Framework.

Libraries provide convenient and reliable implementations of common programming tasks.

Benefits of the Java Collections Framework

The Java Collections Framework provides the following benefits:

  • Reduces programming effort: By providing useful data structures and algorithms, the Collections Framework frees you to concentrate on the important parts of your program rather than on the low-level "plumbing" required to make it work.
  • Increases program speed and quality: This Collections Framework provides high-performance, high-quality implementations of useful data structures and algorithms.
  • Reduces effort to learn and to use new APIs:The standard collection interfaces provide consistency throughout the framework so it takes less time to learn.
  • Fosters software reuse: New data structures that conform to the standard collection interfaces are by nature reusable. The same goes for new algorithms that operate on objects that implement these interfaces.
