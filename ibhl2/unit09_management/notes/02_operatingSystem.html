Role of Operating System

6.1.5 Explain the role of the operating system in terms of managing memory, peripherals and hardware interfaces.

A computer is a system composed of two major components: hardware and software. Computer hardware is the physical equipment. Software is the collection of programs that allows the hardware to do its job. Computer software is divided into two broad categories: the operating system and application programs. Application programs use the computer hardware to solve users' problems. The operating system, on the other hand, manages application programs and controls access to hardware by users.

An operating system has three main functions: (1) manage the computer's resources, such as the cpu, memory, secondary storage, and peripherals, (2) establish a user interface, and (3) execute and provide services for applications software.

Managing Resources

Processor Management

An operating system manages the cpu by providing applications access to its processing capabilities.

In a multitasking operating system more than one program is capable of running at a time. Today, most desktop, laptop, and tablet operating systems function with some type of multitasking operating system.

A multitasking operating system divides the available processor time among the threads or jobs that need it. The operating system allocates a processor time slice to each thread it executes. The currently executing thread is suspended when its time slice elapses, allowing another thread to run. When the system switches from one thread to another, it saves the context of the current thread and restores the saved context of the next thread in the queue.

The length of the time slice depends on the operating system and the processor. Because each time slice is small (approximately 20 milliseconds), multiple threads appear to be executing at the same time. This is actually the case on multiprocessor systems, where the executable threads are distributed among the available processors.

Memory Management

Operating systems use RAM to store the data required for applications and other computer functions to operate. Each computing activity that is ongoing has a certain amount of memory allocated to it, which the operating system manages. In cases where an application attempts to do something that there is not enough memory for, an operating system can sometimes use virtual memory to carry out the required processing, by utilizing a section of hard disk memory and using it as though it is RAM.

Peripheral Management

Peripheral devices are managed by the operating system through the use of drivers. Drivers are special system software that handle communication between the user and the hardware installed or attached to a computer. All input and output operations, although invoked by an application programs, are actually carried out by the operating system.

User Interface

This is the part of the operating system that interacts with the user. There are three different types of user interfaces: command-line, menu-driven, and graphical-user interface.

Application Software Services

The operating system is responsible for executing application software when they are selected by a user. They also provided many services to application software. Here is a list of a few of these services:

  • User Interface API
  • Memory Allocation
  • File Management
  • Error Handling
  • Batch Processing (print jobs)

6.1.7 Outline OS resource management techniques: scheduling, policies, multitasking, virtual memory, paging, interrupt, polling.

Scheduling

Operating system scheduling is the process of controlling and prioritizing messages sent to a processor. An internal operating system program, called the scheduler, performs this task. The goal is maintaining a constant amount of work for the processor, eliminating highs and lows in the workload and making sure each process is completed within a reasonable time frame.

Since nearly every operation on a computer has at least a small amount of processor time involved, the processor can be a major source of slowdowns and bottlenecks. In order to alleviate the strain on the processor, and make sure tasks are completed in a timely manner, most operating systems use some form of task scheduling.

Policies

Policies allow the operating system to implement specific configurations for users and computers. Policies are settings that control how the operating system works. You can use them to customize the user interface, restrict access to certain areas, specify security settings, and much more.

Multitasking

In a multitasking operating system where multiple programs can be running at the same time, the operating system determines which applications should run in what order and how much time should be allowed for each application before giving another application a turn.

Virtual Memory

Virtual memory is a memory management technique that temporarily stores inactive parts of the content in RAM to a disk, restoring it to RAM when quick access to it is needed. This allows software to operate as though the computer has more RAM than it actually does. A computer can utilize virtual memory to store features in a program that are rarely used so they do not take up RAM.

Paging

Paging is a memory management technique where the operating system copies a certain number of pages from a storage device to main memory. When a program needs a page that is not in main memory, the operating system copies the required page into memory and copies another page back to the disk.

With paging the operating system reads data from secondary storage in blocks called pages, all of which have identical size. The physical region of memory containing a single page is called a frame. When paging is used, a frame does not have to comprise a single physically contiguous region in secondary storage. This approach offers an advantage over earlier memory management methods, because it facilitates more efficient and faster use of storage.

Interrupt

An interrupt is an event that alters the sequence in which the processor executes instructions. A peripheral device uses interrupts to notify the CPU when it is ready to send or receive data. Most operating systems today utilize interrupts to control the flow of execution in a computer system.

In general, there are two categories of interrupts: hardware interrupts and software interrupts. A hardware interrupt occurs, for example, when an I/O operation has occurred such as a keystroke on a keyboard or button click on a mouse. A software interrupt occurs when an application program terminates or requests certain services from the operating system.

It is the operating systems job to watch for interrupts. If an interrupt occurs the operating system calls on a program called an interrupt handler that prioritizes the interrupts and saves them in a queue if more than one is waiting to be handled. The operating system then uses a scheduler to figure out which program to give control to next.

Polling

Polling is an alternative to interrupts. In polling a peripheral device is repeatedly checked by the CPU for readiness, and if it is not, the computer returns to a different task.

With polling the processor is responsible for checking peripheral devices to see if they are ready for data transfer. With interrupts the devices are responsible for notifying the processor when they need to transfer data.

Both polling and interrupts have advantages and disadvantages so to be most efficient processors utilize both of them. A good example of polled input is mouse movement. A mouse could produce thousands of interrupts a second when the user moves the mouse. Therefore an operating system will poll the mouse to determine the current mouse position. An example of interrupt-driven input would be a network interface controller (NIC). A NIC has a small memory location called a buffer to accommodate input and output communication. Once the buffer receives a packet of data it requires the CPU to copy that packet to main memory before the next packet arrives and overwrites it. So once a packet is received the NIC will interrupt the CPU so that the data can be removed from the buffer into main memory and signal to the NIC that it's ok to continue receiving data.

A good example of a system that is both polled and interrupt driven is a keyboard. The subsystem that accepts data from the keyboard generally just buffers the data for polling by the CPU but can be programmed to generate interrupts as well. The most famous use of this is the key combination Control-Alt-Delete, which sends an interrupt to the CPU. This is why when a computer is busy or hung pressing this key combination will interrupt the system.

6.1.8 Discuss the advantages of producing a dedicated operating system for a device.

An operating system designed for a single purpose is often called a dedicated operating system. It performs a specific purpose, such as controlling the environment in a building, controlling a gas pump, or opening and closing an automatic door. Dedicated operating systems cannot be used to run other types of programs such as games or business software.

Operating systems that are either found in embedded systems or in firmware embedded in hardware are considered to be dedicated operating systems.

The table below lists advantages and disadvantages of dedicated operating systems.

Advantages Disadvantages
Size - code is written for specific devices therefore it is much smaller than general-purpose operating systems Cost - more expensive to design
Speed - code is smaller and optimized for specific devices so it is much faster Can only be used on the device for which it was designed
Customizable - designed for specific devices therefore can be customized to meet the specific needs of particular devices
The IB curriculum states, "Advantages related to size, speed and customization should be considered. For example, using a dedicated operating system for a cell phone rather than using a pre-existing operating system."
6.1.9 Outline how an operating system hides the complexity of the hardware from users and applications.

The fundamental operation of the operating system is to abstract the hardware to the programmer and user. The operating system provides generic interfaces to services provided by the underlying hardware.

In a world without operating systems, every programmer would need to know the most intimate details of the underlying hardware to get anything to run. Worse still, their programs would not run on other hardware, even if that hardware has only slight differences.

Below are some examples of abstraction that are provided by the operating system:

  • File System - Secondary storage devices such as hard disks, DVDs, and USB drives all work differently at the physical level but in an operating system's virtual world they all appear to be the same. The processes of opening and saving files on these devices is identical (e.g. letters assigned to each device).

  • Application Programming Interface (API) - an operating system provides a programming language called an API that developers of applications use to standardize the features of application programs. One feature provided by the API is a GUI interface that provides controls like buttons, menus, ribbons, toolbars, and sliders that have the same look and feel within different applications. This standardization makes it easier for users to use application programs.

    APIs also abstract processes like printing. The printing process is the same for different application programs. A user does not need to worry about things like what type of printer is connected to a computer. All they have to do is select print from a menu and the operating system takes care of the details.

  • Virtual Memory - gives a user the illusion that they have unlimited memory at their disposal. The details of the swapping of data from the hard drive to RAM is completely hidden from the user.

The IB curriculum states, "Students should be aware of a range of examples where operating systems virtualize real devices, such as drive letters, virtual memory, input devices, the Java virtual machine."
