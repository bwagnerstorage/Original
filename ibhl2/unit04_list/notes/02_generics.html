Generics

Generics is a Java programming language feature that allows a programmer to design a class where the user of the class is able to specify the type of data that it will contain. For example, a user of the ArrayList class instantiates an object using the following format:

ArrayList 
<Integer>
 list = new ArrayList <Integer>();

By using the notation <Integer> the user is able to specify the type of data the ArrayList will contain. This is generics.

Simple Generic Class

Look at the following example below that illustrates how to add generics to a collection class so that it can suppport any type.

public class MyLinkedList<
E
>
{
    private ListNode head;
    private ListNode cur;
    private ListNode prev;
    
    public void insertFirst(
E
 data)
    {
        ListNode newNode = new ListNode(data, null);
        if(head == null)
        {
            head = newNode;
        }
        else
        {
            newNode.setNext(head);
            head = newNode;
        }
        
    }
    
    public void insertLast(
E
 data)
    {
        ListNode newNode = new ListNode(data, null);
        cur = head;
        while(cur != null)
        {
            prev = cur;
            cur = cur.getNext();
        }
        prev.setNext(newNode);
        
    }
    
    public void deleteFirst()
    {
        head = head.getNext();
    }
    
    public void deleteLast()
    {
        cur = head;
        while(cur.getNext() != null)
        {
            prev = cur;
            cur = cur.getNext();
        }
        prev.setNext(null);
    }
    
    public void deleteInside(
E
 key)
    {
       cur = head;
       while(cur != null && !cur.getValue().equals(key))
       {
          prev = cur;
          cur = cur.getNext();
       }    
       if(cur == head && cur.getValue().equals(key))
       {
          System.out.println("test");
          head = head.getNext();
       }
       else if(cur != null)
          prev.setNext(cur.getNext());
    }
    
    public boolean search(
E
 key)
    {
       cur = head;
       while(cur != null)
       {
          if(cur.getValue().equals(key))
             return true;
          cur = cur.getNext();
       }    
       return false;
    }

Note that within the class, the type parameter E is used just like any regular type name. It's used to declare the return type and parameter type for the different methods. Given this class definition, we can use parameterized types such as MyLinkedList<String> and MyLinkedList<Integer>. That is, the MyLinkedList class is used in exactly the same way as built-in generic classes like ArrayList and LinkedList.

 public static void main(String[] args)
 {
     MyLinkedList<String> list = new MyLinkedList<String>();
     list.insertFirst("tiger");
     list.insertFirst("lion");
 }

Note that you don't have to use "E" as the name of the type parameter in the definition of the generic class. Type parameters are like formal parameters in methods. You can make up any name you like in the definition of the class. The name in the definition will be replaced by an actual type name when the class is used to declare variables or create objects.
