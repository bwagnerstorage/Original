Graph

A tree is a hierarchical data structure in which each element or node is connected to one or more nodes beneath it. A tree has the restriction that a child node can only have one parent. A tree is a special type of graph.

In simplest terms a graph consists of elements (normally referred to as vertices or nodes) connected by links (also known as edges). Unlike trees, in graphs, a node can have many parents. Here is an example of a graph.

Below are a list of terms that will help you better understand graphs.

Term Definition
vertex A single node or endpoint in a graph
edge A single connection between two vertices
adjacent Two vertices are adjacent if there is an edge connecting them
connected A graph is connected if every vertex has a path to all other vertices
strongly connected A graph is strongly connected if every pair of vertices has a path between them, in both directions
neighbor Two vertices are neighbors if there is an edge connecting them
path A sequence of edges that can be followed from one vertex to another
simple path A path that contains no repeated vertices
cycle Special kind of path that ends at the same vertex where it originally started
simple cycle A cycle with no repeated vertices except for the first and last one
Hamiltonian cycle A simple cycle that contains all the vertices in the graph
Euler cycle A cycle that contains every edge in the graph exactly once

Classifications of Graphs

There are several common types of graphs

  • Directed or undirected
  • Weighted or unweighted
  • Cyclic or acyclic

Directed or Undirected

Graphs can be classified by whether or not their edges have direction.

  • Undirected Graph: each edge can be traversed in either direction
  • Directed Graph: each edge can be traversed only in a specified direction, typically denoted by an arrow.

Weighted or Unweighted

Graphs can be classified by whether or not their edges have weight.

  • Weighted Graph: edges have a weight assigned to them. This weight typically shows the cost of traversing.
    Example: distances between cities on a map
  • Unweighted Graph: edges have no weight. The edges simply show connections.
    Example: high school course prerequisites

Cyclic or Acyclic

Graphs can be classified by whether or not they have cycles. A cycle is a path through a graph that ends at the same vertex where it originally started.

  • Cyclic Graph: graph contains one or more cycles
  • Acyclic Graph: graph contains no cycles

Graph Problems

Graphs are good in modeling real world problems like representing cities which are connected by roads and finding the paths between cities, modeling air traffic controller system, and representing networks like the Internet. Graphs are also used in social networks like linkedIn, Facebook. For example, in Facebook, each person is represented with a vertex(or node). Each node is a structure and contains information like person id, name, gender, locale etc.

Recognizing a Graph Problem

The first key to solving a graph related problem is recognizing that it is a graph problem. If you see the problem involves a grid or network of some kind that can be a clue that it might be a graph problem. Secondly, if you are required to find a path of any sort, it is usually a graph problem as well. Finally, some common keywords associated with graph problems include: vertices, nodes, edges, connections, connectivity, paths, cycles and direction. An example of a description of a simple problem that exhibits some of these characteristics is shown below.

Sample Graph Problem

Kolt has moved to a new town and he has to walk to his new school. His school is directly behind his house but there is a 100 acre patch of woods between them. Kolt has decided to see if he can find a shortcut to school through the woods so it does not take him as long to get to school. He found his house on Google Maps and noticed that the patch of woods is in a square shape and there is a creek along one side of the woods and a road on the other.

Write a program that simulates this scenario. Given a 10 x 10 2-dimensional grid marked with the letters 'H', 'X', 'T', 'C', 'R' and a '.'. The 'H' represents Kolt's house, the starting point. The 'X' is his school, the ending point. 'T' represents trees, the 'C' represents the creek, the 'R' represents the road, and the '.' represents open areas were he can walk. He can walk horizontally, vertically, and diagonally. Find the minimum number of steps it takes Kolt to find a path from his house to his school. There exists at least one path through the woods.

This is an example of a shortest path problem which is one type of problem graphs are often used to solve.
