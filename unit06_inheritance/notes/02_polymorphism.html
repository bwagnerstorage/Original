Polymorphism

In Java, polymorphism manifests itself in two forms. The first form, method overloading, we have already explored. Method overloading is when more than one method is defined within a single class with the same name. The add methods in the ArrayList class are an example of method overloading. 

list.add("friend");        // append
list.add(5, "friend");     // insert

The second use of polymorphism in Java is best explained through an example.

I start by defining a class named Pet. For simplication, its variables and methods are not shown.

class Pet
{

}

Next I define two classes that are subclasses of Pet named Dog and Cat. Again their variables and methods are not shown.

class Dog extends Pet
{

}

class Cat extends Pet
{

}

Now here comes the polyporphism. I create some client code that uses all three classes.

Pet pet;
pet = new Dog();
pet = new Cat();

I declared a variable of type Pet. Notice that I can assign both a Dog object and a Cat object to variable pet. This is polyporphism. The pet variable can reference any object of a class that is a subclass of Pet.

Here is another example.

public method print(Pet pet)
{
   System.out.println(pet);   // call toString method
}

I have defined a method named print that is passed a parameter of type Pet. What type of Pet can I print in this method? You guessed it. A Dog and a Cat or any other object of a class that is a subclass of Pet. That's Polymorphism!

Required Cast When Using Polymorphism

Suppose in our Pet class I define an instance variable, a constructor, and a method.

class Pet
{
    private String petName;
	
    public Pet(String name)
    {
        petName = name;
    }
	
    public String getName()
    {
       return petName;
    }
}

In the Dog class I define a constructor and a method.

class Dog extends Pet
{
   public Dog(String name)
   {
      super(name);
   }
   
   public void speak()
   {
      System.out.println("ruff ruff");
   }
}

In the client, I call the print method and pass it a dog object with the following code.

Dog dog = new Dog("Maddie");
print(dog);

I modify the print method by adding the following code.

public method print(Pet pet)
{
   System.out.println(pet.getName());
   
((Dog) pet)
.speak(); }

Even though the object passed to the print method is a Dog the pet variable can only access Pet methods, therefore the call to the getName method works fine, but the call to the method speak requires the pet variable be cast to a Dog before the call is made. Subclass methods can only be seen by superclass variables with a cast. Of course casting a pet variable to Dog when it actually stores a Cat would cause an exception.
