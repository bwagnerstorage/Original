UUnordered Collection: Map

Interfaces: Map

Implementing Classes: HashMap, TreeMap

Map

A map is a collection of key-to-value mappings, where both key and value are objects. A map cannot contain duplicate keys, which means that each key maps to exactly one value. Different keys, however, can map to the same value. If a duplicate key is put into map the previous value for that key will be replaced with new value. Two examples of maps are a dictionary (key - word, value - definition) and a phone book (key - name, value - phone number).

The JCF provides an interface with the following methods

interface java.util.Map<K,V>

int size() returns the number of key/value mappings in the map
boolean containsKey(Object key)

returns true if the map contains a mapping for key, false otherwise
put(K key, V value) Associates key with value
returns the value formerly associated with key
V get(Object key) returns the value associated with key or null if there is no associated value
V remove(Object key) removes and returns the value associated with key
Set<K> keySet() returns the set of keys contained in the map

Implementing Classes

The two Map implementations in the AP subset are HashMap and TreeMap. Once again their names are derived from the technique used to store and retrieve the elements in a map.

HashMap (class java.util.HashMap<K,V> implements java.util.Map<K,V>)

The HashMap class implements the Map interface with a hash table which means there is no particular ordering of elements. HashMap provides O(1) run times for the get and put operations.

The methods of HashMap in the AP Java subset are the same methods as those given for the Map interface: put, get, remove, containsKey, size, and keySet.

Iterating over Maps

To iterate through a map use the keySet method. The keySet method of the Map interface returns the set of keys for the map. You can use code like the following to iterator over the set of keys:

Iterator iter = m.keySet().iterator;  // get set of keys
while(iter.hasNext())
{
   System.out.println(iter.next());
}

HashSet Examples

import java.util.*;

public class MapDemo
{
    public static void main(String[] args)
    {
       // constructor
       HashMap<Integer, String> map = new HashMap<Integer, String>();
       
       // put method
       map.put(1, "dog");
       map.put(2, "cat");
       map.put(3, "bird");
       
       // get method
       String ans1 = map.get(1);
       String ans2 = map.get(2);
       String ans3 = map.get(3);
       
       System.out.println(ans1);
       System.out.println(ans2);
       System.out.println(ans3);
       
       // remove method
       String ans4 = map.remove(2);
       System.out.println(ans4 + " has been removed");
       
       // size method
       System.out.println("Size = " + map.size());
       
       // containsKey method
       if(map.containsKey(1))
       {
          System.out.println("key is in map");
       }
       
       // Iterate over map using set of keys
       Iterator iterator = map.keySet().iterator();
       while(iterator.hasNext())
       {
          System.out.println(map.get(iterator.next()));
       }
       
    }
}

TreeMap (class java.util.TreeMap<K,V> implements java.util.Map<K,V>)

The TreeMap class implements the SortedMap interface using a balanced binary search tree. The class guarantees ascending key order based on the ordering of the key class. TreeMap guarantees O(log n) performance for the containsKey, get, and put operations.

The operations for the TreeMap are also those described for the Map interface: put, get, remove, containsKey, size, and keySet.

TreeMap Example
