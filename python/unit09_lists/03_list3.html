Static vs Dynamic

We have seen how lists can be created in a static context where the items stored in the list are assigned values within the code itself. For example, 

names = ["John", "Maria", "Amy", "Joe", "Juan"]

assigns five names to the list when it is created. This static approach can be good for situations were there is a small set of data. However, lists can also store large amounts of data where it would not be practical to enter the data manually. In this case, programmers use a more dynamic approach that involves assigning values to a list while a program is running. This input can be from an input device such as a keyboard or data stored in a file on a secondary storage device. This dynamic approach involves using List methods like append, insert, and remove. The sample code below, demonstrates how to populate a list where the data comes from user input. 

cities = []   #empty list
i = 0;  # loop control variable
size = int(input("How many cities would like to enter-->"))
while i < size:
   name = input(str(i+1) + ". Enter a city name -->")
   cities.append(name)
   i+=1
      
print()
print(cities)

Lab Exercise

Your friend Joe has decided to get his life in order, so he has decided to start by creating a ToDo List each day so that he does not waste time doing things that don't really accomplish anything. He wants his list to be prioritized so that the tasks that are the most important are at the top of the list. He knows that you have been learning Python so he has asked you to write a program that can create a TODO List.

  1. Create an empty List named taskList

  2. Create a sentinel-controlled while loop that continues looping until the it encounters the sentinel values "N" or "n". The user should be able to type their response in upper or lower case letters.

  3. Prompt the user to enter a task that needs to be done.

  4. Prompt the user for a priority level by typing H for high level and L for low level. The user should be able to type their response in upper or lower case letters.

  5. If the priority level is High, then use the List class's insert method to add the new task to the front of taskList. IF the priority level is low, then use the List class's append method to add the new task to end of taskList.

  6. Use a while loop to display the taskList. The tasks in the taskList should be numbered from 1 to the taskList - 1. Include an appropriate banner on top of the list. This TODO list should be displayed each time a new task is added to the list.

  7. Prompt the user to see if they wish to enter another task. The user's response should be one of the sentinel values: "Y" or "y" and "N" or "n".

  8. If the sentinel values "Y" or "Y" were entered the while loop should continue, otherwise the program should exit.
