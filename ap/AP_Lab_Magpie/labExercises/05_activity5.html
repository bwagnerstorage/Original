Magpie Lab - Activity 5

Introduction

When you last worked with the Magpie, default responses were handled with a nested if statement. This certainly worked, and you could add more responses, but it was a bit awkward. An easier way to keep track of default responses is with an array. In this activity, you will see how an array makes handling default responses much easier.

Prepare

You are going to use the Activity5 project in this activity. You will need to make Activity5 the active project. This can be accomplished by right-clicking on the project name and selecting Sets as Active Project.

Exploration

Run this version of the Magpie class. You should see no difference in its outward behavior. Instead, it has been changed so that its internal structure is different. This is called code refactoring. That's one of the big benefits of dealing with methods as black boxes. As long as they perform the action required, the user does not care about how they perform the action.

Exercise

Notice at the the top of the Magpie5 class an array of strings named RANDOM_RESPONSES has been defined. The definition includes the use of the keywords "static" and "final". Recall that when you declare a variable with final modifier the variable becomes a constant and it data cannot be altered. The will cover the static modifier later. The reason for including the static and final modifiers is to improve the efficiency of the array.

private static final String[] RANDOM_RESPONSES = {
   "Interesting, tell me more",
   "Hmmm.",
   "Do you really think so?",
   "You don't say."
};

Using an intializer list {} the array includes 4 random responses.


  • Your first task is to add four more responses to the array.

Method getRandomResponse

The random responses are generated in a method named getRandomResponse. It is declared at the very bottom of the class. You are going to implement this method.

Generating Random Numbers

Recall that one way to create a random number in Java is to use the Math class's random method. Look at the following code segment:

int r = (int)(Math.random() * 8);

This code will generate a random number in the range of 0 to 7.

Array length Variable

Look at the next code segment that is a slight modification to the one above.

int r = (int)(Math.random() * RANDOM_RESPONSES.length;

Recall that an array has access to a variable named length that keeps track of the size of the array. By replacing the number 8 with the name of the array and the length variable we are able to generate a random number in the range of 0 to length of the array - 1. For example, if the array is size 10 then the random numbers generated would be in the range of 0 to 9 which corresponds to indexing numbers use to access the contents of the array.
