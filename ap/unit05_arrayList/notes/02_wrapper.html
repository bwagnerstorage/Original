The ArrayList is a powerful data structure. There is, however, one major drawback. Unlike an array, an ArrayList can contain only objects, not primitive data types. You cannot use primitive types as type parameters - there is not an ArrayList<int> or ArrayList<double>.

Wrapper Classes

Since numbers are not objects in Java, you cannot directly insert them into ArrayList. To store sequences of numbers in an ArrayList, you must turn them into objects by using wrapper classes. There are wrapper classes for all eight primitive types:

Primitive Data Type Wrapper Class
boolean Boolean
char Character
byte Byte
short Short
int Integer
long Long
float Float
double Double

Note that the wrapper class names start with uppercase letters, and that two of them differ from the names of the corresponding primitive types: Integer and Character.

Each wrapper class object contains a value of the corresponding primitive type. For example, you can create an object of the wrapper class Double with the following line of code:

Double d = new Double(29.95);

The object created contains a value of type double. Refer to illustration below.

Wrapper objects can be used anywhere that objects are required instead of primitive type values. For example, you can collect a sequence of floating-point numbers in an ArrayList <Double>.

Auto-boxing

Conversion between primitive types and the corresponding wrapper classes is automatic. This process is called auto-boxing.

For example, if you assign a number to a Double object, the number is automatically "put into a box", namely a wrapper object.

Double d = 29.95;  // auto-boxing; same as Double d = new Double(29.95);

Wrapper objects are automatically "unboxed" to primitive types.

double x = d;      // auto-boxing

Auto-boxing even works inside arithmetic expressions. For example, the statement

Double e = d + 1;

is perfectly legal. It means

Auto-unbox d into a double 
Add 1 
Auto-box the result into a new Double 
Store a reference to the newly created wrapper object in e

ArrayLists of Numbers

ArrayLists of numbers are straightforward. Simply remember to use the wrapper type when you declare the ArrayList, and then rely on auto-boxing.

ArrayList<Double> list = new ArrayList<>();
list.add(29.95);
double x = list.get(0);

With older versions of Java, using wrapper classes to store numbers in an ArrayList was a considerable hassle because you had to manually box and unbox the numbers.
