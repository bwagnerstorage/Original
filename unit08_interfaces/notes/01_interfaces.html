Interfaces

In Java an interface is a set of methods that classes must incorporate into their design. Interfaces are similar to abstract classes in that they both contain abstract methods. They are different, however, in that interfaces can only contain abstract methods and constants. They do not contain variables, constructors, and implemented methods.

Interfaces are not classes.
Look at the following example of an interface. 
public 
interface
 Pet
{
   public abstract void eat();
}

To define an interface you use the keyword interface followed by the name of the interface. The methods declared in an interface must be both public and abstract. In fact, if you do not include these modifiers the compiler will add them for you. For this reason, most programmers do not include them.

public 
interface
 Pet
{
   void eat();
}

Interfaces can also contain constants.

public 
interface
 Pet
{
   public final int NUM_MEALS = 2;
   
   void eat();
}

Again the public declaration for the constant is optional.

Implementing Interfaces

Interfaces are implemented by classes. Look at the Dog class below that implements the Pet interface.

public class Dog 
implements
 Pet
{   
   public void eat()
   {
       // code to eat
   }
}

To implement an interface a class uses the keyword implements.

Since the Dog class implements the Pet interface, it is required to implement the eat method.

The Cat class below also implements the Pet interface therefore it also implements the eat method.

public class Cat 
implements
 Pet
{   
   public void eat()
   {
       // code to eat
   }
}

Since interfaces are not classes they cannot be instantiated. The following code would cause a compile error.

Pet pet = new Pet();   
// error

Purpose of Interfaces

Interfaces are used in Java to establish a common interface among related classes. Implementing an interface allows a class to become more formal about the behavior it promises to provide. Interfaces form a contract between the class and the outside world. If your class claims to implement an interface, all methods defined by that interface must appear in its source code before the class will successfully compile.

In the example above, both the Dog and Cat classes implement the Pet interface, therefore clients of these classes know they both contain an eat method.

Implementing Multiple Interfaces

When using inheritance in Java a class can only extend one parent class. Multiple inheritance is not allowed. Interfaces, however, are not classes and multiple implementations are allowed. As an example, the Dog class below now implements both a Pet and Security interface.

public class Dog implements Pet, Security
{   
    
}

Each interface is separated by a comma.

Classes can also implement interfaces in conjunction with extending a class.

public class Dog extends Animal implements Pet, Security
{   
    
}
